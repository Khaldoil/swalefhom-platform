# دليل نظام الإشعارات بالبريد الإلكتروني

## نظرة عامة

تم إضافة نظام إشعارات بريد إلكتروني شامل لموقع سواليفهم يقوم بإرسال إشعارات تلقائية لجميع التحديثات والأحداث المهمة في الموقع.

---

## الميزات المضافة

### 1. إشعارات قبول/رفض طلبات السفراء

**الموقع:** صفحة طلبات السفراء (`/admin/applications`)

**الوظيفة:**
- عند قبول أو رفض طلب سفير، يتم إرسال:
  - بريد إلكتروني للمتقدم يُعلمه بالقرار
  - بريد إلكتروني للمدير يؤكد إتمام العملية

**كيفية الاستخدام:**
1. افتح صفحة طلبات السفراء
2. اضغط على زر القبول ✓ أو الرفض ✗
3. ستفتح نافذة تطلب منك:
   - تأكيد القرار
   - إضافة ملاحظة اختيارية للمتقدم
4. اضغط "إرسال" لتنفيذ القرار وإرسال الإشعارات

**محتوى البريد الإلكتروني:**
- **للمقبولين:** رسالة تهنئة + شرح الخطوات القادمة
- **للمرفوضين:** رسالة تشجيعية + إمكانية التقديم مرة أخرى
- **للمدير:** ملخص القرار + معلومات المتقدم

---

### 2. إشعارات نشر/رفض القصص

**الموقع:** صفحة القصص الإدارية (`/admin/stories`)

**الوظيفة:**
- عند نشر أو رفض قصة، يتم إرسال:
  - بريد إلكتروني للراوي يُعلمه بالقرار
  - بريد إلكتروني للمدير يؤكد إتمام العملية

**كيفية الاستخدام:**
1. افتح صفحة إدارة القصص
2. اضغط على زر النشر ✓ أو الرفض ✗ للقصص بحالة "بانتظار المراجعة"
3. ستفتح نافذة تطلب منك:
   - تأكيد القرار
   - إضافة رسالة اختيارية للراوي
4. اضغط "إرسال" لتنفيذ القرار وإرسال الإشعارات

**محتوى البريد الإلكتروني:**
- **للمنشورة:** تهنئة + رابط للقصة المنشورة
- **للمرفوضة:** توضيح السبب (إذا تم إدخاله) + تشجيع على التحسين
- **للمدير:** ملخص القرار + معلومات القصة والراوي

---

### 3. إشعارات المحتوى الجديد التلقائية

**الوظيفة:**
يتم إرسال إشعار تلقائي للمدير عند إضافة:
- تدوينة جديدة
- كتاب جديد في المكتبة
- فعالية جديدة
- دورة تدريبية جديدة

**الإرسال:**
- يحدث تلقائياً عند إنشاء المحتوى من لوحة التحكم
- لا يحتاج لأي إجراء إضافي من المدير

**محتوى البريد الإلكتروني:**
- عنوان المحتوى
- النوع (تدوينة، كتاب، فعالية، تدريب)
- تفاصيل أساسية
- رابط مباشر للوحة التحكم

---

### 4. دالة الإشعارات العامة

**الموقع:** Edge Function (`/supabase/functions/notify-admin`)

**الوظيفة:**
دالة عامة يمكن استخدامها لإرسال أي نوع من الإشعارات للمدير.

**أنواع الإشعارات المدعومة:**
- `new_story`: قصة جديدة
- `new_application`: طلب سفير جديد
- `new_blog`: تدوينة جديدة
- `new_gallery`: محتوى معرض جديد
- `new_event`: فعالية جديدة
- `new_training`: دورة تدريبية جديدة
- `new_pioneer`: رائد تراث جديد
- `system_alert`: تنبيه نظام

**مستويات الأولوية:**
- `high`: عاجل (أحمر)
- `medium`: مهم (برتقالي)
- `low`: معلومة (أخضر)

## Edge Functions المضافة

### 1. notify-ambassador-decision
**المسار:** `/supabase/functions/notify-ambassador-decision`

**الغرض:** إرسال إشعارات قرارات السفراء (قبول/رفض)

**البارامترات:**
```json
{
  "application": {Object}, // بيانات المتقدم
  "status": "approved | rejected",
  "adminNote": "string" // ملاحظة اختيارية
}
```

---

### 2. notify-story-decision
**المسار:** `/supabase/functions/notify-story-decision`

**الغرض:** إرسال إشعارات قرارات القصص (نشر/رفض)

**البارامترات:**
```json
{
  "story": {Object}, // بيانات القصة
  "status": "published | rejected",
  "adminNote": "string" // رسالة اختيارية
}
```

---

### 3. notify-admin
**المسار:** `/supabase/functions/notify-admin`

**الغرض:** إرسال إشعارات عامة للمدير

**البارامترات:**
```json
{
  "type": "نوع الإشعار",
  "title": "عنوان الإشعار",
  "data": {Object}, // بيانات الإشعار
  "priority": "high | medium | low" // اختياري
}
```

**مثال على الاستخدام:**
```javascript
await fetch(`${supabaseUrl}/functions/v1/notify-admin`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${supabaseAnonKey}`,
  },
  body: JSON.stringify({
    type: 'new_blog',
    title: 'تدوينة جديدة: عنوان التدوينة',
    data: {
      title: 'عنوان التدوينة',
      category: 'التصنيف',
      excerpt: 'مقتطف من التدوينة'
    },
    priority: 'medium'
  }),
});
```

## نصائح الاستخدام

### للمدير:

1. **راجع الملاحظات قبل الإرسال:**
   - الملاحظات المرسلة تظهر للمستلم
   - استخدم لغة مهنية ومشجعة

2. **استخدم الملاحظات بحكمة:**
   - **للقبول:** عبّر عن الحماس والترحيب
   - **للرفض:** قدم توجيهات بناءة وتشجيع

3. **راقب صندوق بريدك:**
   - ستصلك نسخ من جميع الإشعارات المرسلة
   - يمكنك متابعة جميع الإجراءات المتخذة

4. **في حالة فشل الإرسال:**
   - سيتم تحديث الحالة في قاعدة البيانات رغم فشل الإشعار
   - ستظهر رسالة تحذير في الواجهة
   - تحقق من إعدادات البريد الإلكتروني

## استكشاف الأخطاء

### لم يصل البريد الإلكتروني:

1. تحقق من صندوق البريد العشوائي (Spam)
2. تأكد من أن البريد الإلكتروني المسجل صحيح
3. راجع سجلات Edge Functions في Supabase
4. تحقق من حصة Resend API المتبقية

### رسالة خطأ عند الإرسال:

1. تحقق من اتصال الإنترنت
2. راجع متغيرات البيئة (.env)
3. تأكد من أن Edge Functions تعمل
4. راجع console في المتصفح للمزيد من التفاصيل

## التطويرات المستقبلية المقترحة

1. **إشعارات SMS:** إضافة خيار لإرسال رسائل نصية للقرارات المهمة
2. **قوالب قابلة للتخصيص:** إمكانية تخصيص قوالب البريد من لوحة التحكم
3. **إحصائيات الإشعارات:** تقارير عن معدلات الفتح والنقر
4. **إشعارات متعددة اللغات:** دعم اللغة الإنجليزية للمستخدمين الدوليين
5. **جدولة الإشعارات:** إمكانية جدولة إرسال الإشعارات في وقت محدد

## الدعم الفني

في حالة وجود أي مشاكل أو استفسارات، يرجى:
- مراجعة هذا الدليل أولاً
- فحص سجلات Edge Functions في Supabase
- التواصل مع فريق التطوير

**ملاحظة:** هذا النظام يعمل بشكل تلقائي ولا يحتاج لصيانة دورية. جميع الإشعارات تُسجل في قاعدة البيانات ويمكن مراجعتها في أي وقت.
